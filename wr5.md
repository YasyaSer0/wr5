## 1. Робота ОС Linux з периферійними пристроями

Під час роботи з персональним комп’ютером користувачі досить часто підключають різні периферійні пристрої, такі як флеш-накопичувачі, принтери, клавіатури, миші та інше обладнання. Для взаємодії з такими пристроями операційна система Linux використовує спеціальний механізм, який ґрунтується на роботі драйверів, файлової системи та системних служб керування пристроями.

У операційній системі Linux усі пристрої відображаються у вигляді спеціальних файлів, що знаходяться в каталозі /dev. Коли до комп’ютера підключається новий пристрій, ядро операційної системи автоматично розпізнає його та створює відповідний запис у системі.

Наприклад, після підключення флеш-накопичувача система визначає його як диск, наприклад /dev/sdb або /dev/sdb1. Після цього пристрій може бути змонтований у файлову систему, що дозволяє користувачеві отримати доступ до файлів, які зберігаються на флешці.

Для роботи з принтерами в операційній системі Linux використовується спеціальна система друку CUPS (Common Unix Printing System). Вона відповідає за виявлення принтерів у системі, встановлення необхідних драйверів, керування чергою друку та надсилання документів на друк. Користувач може виконувати друк як за допомогою графічного інтерфейсу, так і через термінал, використовуючи відповідні команди, наприклад lp.

---

### Суть операції монтування

Монтування - це процес підключення файлової системи зовнішнього пристрою до основної файлової системи операційної системи Linux.

У Linux усі файли та пристрої знаходяться в єдиній ієрархічній файловій системі, тому для доступу до даних на флешці або іншому носії необхідно спочатку змонтувати його в певну директорію. Зазвичай флеш-накопичувачі автоматично монтуються у каталоги типу:
```bash
/media/username/
```
або
```bash
/mnt/
```
Після монтування користувач може відкривати файли на пристрої, копіювати їх, редагувати або видаляти. Монтування може виконуватися автоматично через графічний інтерфейс або вручну за допомогою команди:
```bash
mount
```
Таким чином, операція монтування дозволяє інтегрувати зовнішній носій у файлову систему Linux і забезпечує доступ до його вмісту.

### Різниця роботи з периферією у Linux та Windows

Між операційними системами Linux та Windows існують певні відмінності у роботі з периферійними пристроями.

У системі Windows підключення пристроїв відбувається більш автоматично. Наприклад, після підключення флеш-накопичувача система сама призначає йому букву диска, наприклад E: або F:, після чого користувач одразу може працювати з файлами через провідник.

У Linux використовується трохи інший принцип. Тут пристрої відображаються у вигляді спеціальних файлів, які знаходяться у каталозі /dev. Щоб отримати доступ до даних на підключеному пристрої, необхідно виконати операцію монтування, після якої пристрій стає доступним у певній папці файлової системи.

Крім того, у Linux багато операцій можна виконувати через термінал за допомогою спеціальних команд, таких як mount, cp, lsblk, lp та інших. У Windows більшість дій користувачі виконують через графічний інтерфейс.

Отже, Linux надає більше можливостей для керування пристроями через командний рядок, тоді як у Windows робота з периферійними пристроями переважно здійснюється за допомогою графічного інтерфейсу.
